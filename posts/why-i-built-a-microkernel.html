Why I Built a Micro-kernel from Scratch

July 2026

If you've ever wondered what happens between pressing the power button and seeing your desktop — that magical space where hardware meets software — you've stumbled into the territory of operating systems. And somewhere in that territory, I decided to build my own.

The Beginning

It started with xv6, a teaching operating system from MIT. Reading its code was a revelation — suddenly, concepts like process scheduling, page tables, and system calls weren't abstract textbook ideas anymore. They were real, tangible pieces of code that I could read, modify, and break.

But xv6 is a monolithic kernel. Everything — the file system, the process manager, the device drivers — lives in kernel space. I wanted to understand the alternative: the microkernel architecture, where the kernel does as little as possible and everything else runs as user-space servers communicating through message passing.

What COSMOS Does

COSMOS is my attempt at a MINIX-inspired microkernel. Here's the architecture:

  • Kernel: Handles only interrupts, IPC (inter-process communication), and basic scheduling
  • Process Manager: Runs in user space, manages process lifecycle
  • File Server: A virtual file system, also in user space
  • Memory Manager: Handles page allocation and virtual memory

The key insight? The kernel becomes tiny and auditable. Each server can crash independently without bringing down the whole system. It's elegant — and incredibly hard to get right.

What I Learned

Building an OS teaches you things no textbook can:

The hardest bugs are the ones where the symptom is a thousand instructions away from the cause.

I learned to think in terms of privilege levels, to debug with nothing but printf statements and raw memory dumps, and to appreciate the sheer engineering that goes into making Linux feel "simple" to use.

Should You Try This?

If you're a CS student — absolutely. Start with xv6, read the xv6 book, and then go deeper. You'll never look at your computer the same way again.

The COSMOS source code is available on GitHub. Feel free to explore, break things, and send me questions.
